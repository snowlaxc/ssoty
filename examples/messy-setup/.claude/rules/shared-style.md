# Shared Style (acme-corp)

All agents at acme-corp must keep diffs small and reviewable, write a regression test before refactoring untested code, prefer deletion over new abstractions, and never add a dependency without explicit approval. Every commit message explains why the change was made, not merely what changed.

See `team-rules.md` for team formation defaults and `meta-layout.md` for the
rules directory layout.
